# 食事管理AIでジム収益を月+8万円増やす

パーソナルジムの差別化と収益化の最適解が**食事管理AI**です。1人あたり月+11,000円の追加収益を、ほぼゼロ稼働で得られる仕組みを解説します。

## 食事管理AIの仕組み

```
顧客が食事写真をLINEで送る
   ↓
AI（Claude）が画像を分析
   ↓
カロリー・栄養素を自動算出
   ↓
個別アドバイスを生成
   ↓
顧客に返信（オーナーは確認のみ）
```

**1顧客あたりの対応時間：1日平均3〜5分**。10人の顧客でも月10〜15時間で済みます。

## 必要なツール

| ツール | 用途 | コスト |
|---|---|---|
| Claude（API or Pro） | 画像分析・アドバイス生成 | 月3,000円〜 |
| LINE公式アカウント | 顧客とのやり取り | 無料〜 |
| Google スプレッドシート | 顧客データ管理 | 無料 |
| Notion or Airtable | 食事記録ストック | 無料〜 |

**初期投資：実質0円。月のランニングコスト：5,000円以内**。

## 顧客への提供パッケージ

### 標準プラン（11,000円/月）

| サービス内容 | 提供方法 |
|---|---|
| LINE食事報告 | 1日3食まで写真で報告可 |
| AI即時フィードバック | 24時間以内に返信 |
| 週次振り返り | 週末に1週間のサマリーを送付 |
| 月次レポート | 体重・体脂肪・食事傾向を分析 |
| トレーナーレビュー | 月1回オーナーが直接コメント |

**顧客視点では「24時間サポートしてもらえる安心感」**が最大の価値。

## プロンプト設計（最重要）

### 食事画像分析の標準プロンプト

```
以下の食事画像を分析してください。

【顧客情報】
- 年齢：42歳女性
- 目標：3ヶ月で5kg減量
- 1日摂取カロリー目安：1,400kcal
- 既往歴：なし
- 好み：和食中心

【出力フォーマット】
1. 推定カロリー（◯kcal）
2. PFC バランス（タンパク質◯g・脂質◯g・炭水化物◯g）
3. 良い点（1〜2個）
4. 改善ポイント（1〜2個）
5. 次の食事への提案（1個）

【トーン】
親しみやすく、ダメ出しではなく前向きに。100文字以内。
```

このプロンプトをLINE Bot連携で自動化すれば、ほぼリアルタイムで返信が返ります。

### 週次振り返りの自動生成

```
以下の1週間の食事記録から振り返りを作ってください。

【食事記録】
[7日分の食事ログをデータで渡す]

【出力】
1. 1週間の総カロリー（推定）
2. PFCバランスの傾向
3. 良かった日・改善が必要な日
4. 来週の改善ポイント3つ

【字数】400文字以内
```

## 実際の収益インパクト

### 8人の顧客に展開した場合

| 項目 | 金額 |
|---|---|
| 月額料金 | 11,000円 × 8人 = 88,000円 |
| AI利用コスト | −5,000円 |
| 自分の稼働時間 | 月15時間 |
| **実質収益** | **約83,000円/月** |

**時間単価換算：約5,500円/h**。指導と並行して提供できるため、純粋な追加収益となります。

## 食事管理AIの限界と注意点

### AIに任せきれない部分

- **アレルギー対応**：必ずトレーナーが確認
- **疾患・薬との相互作用**：医療領域は触れない
- **メンタルケア**：継続できない理由のヒアリングは人がやる
- **モチベーション管理**：人間関係でしか解決できない

### 法的注意点

- **医療行為に該当する助言は避ける**（医師法違反のリスク）
- **「治る」「効く」等の断定表現は使わない**（薬機法）
- **AI生成と明示する**（顧客への透明性）

## 顧客満足度を上げる3つのコツ

1. **返信スピード**：AI活用でも30分〜2時間以内の返信を維持
2. **トレーナーの一言を添える**：完全自動化はしない
3. **月1回は対面or電話で振り返り**：人間関係を維持

## 導入の手順

### 1ヶ月目：プロトタイプ構築

- Claude APIキーを取得
- LINE公式アカウントを設定
- プロンプトを5パターン設計
- 既存顧客1〜2名で試運転

### 2ヶ月目：本格運用開始

- 既存顧客全員にオプション案内
- 加入率50%を目指す
- フィードバックを集めて改善

### 3ヶ月目：拡張・販売

- 他事業者へのコンサル提案開始
- システムをマニュアル化
