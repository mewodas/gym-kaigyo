# 記事自動生成プロンプト

このファイルは、ジム開業ラボの記事を自動生成するためのプロンプトテンプレートです。Claude Code の cron / scheduled routine から参照されます。

## 自動実行の手順

1. `pipeline/topics.json` を読む
2. `queue` 配列から `status: "pending"` の最初の項目を取得
3. その項目の情報をもとに記事をMDX形式で生成
4. `content/{category}/{slug}.mdx` に書き込む
5. `topics.json` の該当項目の `status` を `"published"` に変更
6. `git add . && git commit -m "feat(content): add {slug}" && git push` を実行

## 記事の品質基準

- **文字数**: 2,500〜4,000文字
- **トーン**: 実体験ベース・具体的な数字を入れる・押し売り回避
- **必須要素**:
  - リード文（200文字程度で記事の概要と価値を伝える）
  - H2見出しでの構造化（5〜7個程度）
  - 表（テーブル）を1〜3個含める
  - 失敗例 or 注意点
  - 「まとめ」セクション
  - 数字・固有名詞・実例を含める

## frontmatter の必須項目

```yaml
---
title: "..."         # topics.json の title をそのまま使う
description: "..."   # topics.json の description をそのまま使う
date: "YYYY-MM-DD"   # 実行日
tags: ["...", "..."] # topics.json の keywords をそのまま使う
---
```

## 文体の例

- 「実際に開業したオーナーの視点で...」
- 「筆者の経験では...」
- 「実例として、私が運営しているジムでは...」
- 客観的・実用的・押し売り回避

## NG表現（薬機法・景品表示法）

- 「絶対に痩せる」「治る」「効果保証」
- 「最も○○」「業界No.1」（根拠のない場合）
- 過度な煽り表現

## 生成テンプレート（参考）

```
---
title: "{title}"
description: "{description}"
date: "{date}"
tags: [...keywords]
---

# {見出し}

{リード文：200文字程度。なぜこの記事が役立つか、誰向けかを明確に。}

## {H2セクション1：問題提起 or 結論先出し}

{本文。300〜500字。}

## {H2セクション2：具体的な情報・データ}

{表や箇条書きを含める。}

| 項目 | 数値 |
|---|---|
| ... | ... |

## {H2セクション3：手順 or 詳細}

...

## {H2セクション4：実例・失敗例}

...

## まとめ

{記事の要点を3〜4文でまとめる。}
```

## トピック追加ルール

新しいトピックを追加する場合は、`queue` 配列の末尾に追加。`status` は `"pending"` で初期化。
